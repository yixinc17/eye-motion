## 二、分区判定逻辑

| 区域 | 条件 | 眼目标 | 头目标 | 说明 |
|------|------|--------|--------|------|
| **死区** | \|nx_s\| < 0.12 且 \|ny_s\| < 0.12 | 0 | 0 | 中心小方块，不跟随 |
| **眼区** | 0.12 ≤ \|nx_s\| < 0.75 | nx_s × eyeGain × eyeMaxPx | 0 | 仅眼动，头保持 |
| **头区** | \|nx_s\| ≥ 0.75 | 见协同四阶段 | 见协同四阶段 | 眼+头协同 |

## 四、眼-体协同机制逻辑详解

### 4.1 设计理念

模拟**人眼生理特性**：眼动快、头动慢。目标在视野边缘时，先快速转动眼睛，确认目标稳定后再转头。

### 4.2 旋转后坐标（effective）核心公式

```
effective_nx = nx_s - headYaw / headMaxDeg
effective_ny = ny_s   （水平旋转不影响垂直）
```

**含义**：头转动后，目标在「头坐标系」中的位置。

- 头转向目标时，目标在视野中更居中 → `effective_nx` 趋近 0
- 当 `|effective_nx| < E_exit (0.25)` 时，认为**头已到位**，眼可归位按正常跟随

### 4.3 四阶段与代码对应

| 阶段 | coordPhase | 眼目标 | 头目标 | 判定条件 |
|------|------------|--------|--------|----------|
| ① 眼先动 | `eye_first` | ±eyeMaxPx | headYaw（不变） | 进入头区，timer < 1000ms |
| ② 延迟 | `eye_first` | ±eyeMaxPx | headYaw（不变） | timer < 1000ms，**headYaw 不更新** |
| ③ 头转 | `head_turn` | ±eyeMaxPx | nx_s × headGain × headMaxDeg（headGain=1 时转到人脸居中） | timer ≥ 1000ms，\|effective_nx\| ≥ 0.25 |
| ④ 眼归位 | `eye_return` | effective × eyeMaxPx | 继续向目标 | \|effective_nx\| < 0.25 |

### 4.4 关键实现细节

1. **延迟期间头不更新**（sketch.js 159-161 行）：
   ```javascript
   if (!(p.zone === 'head' && p.head_delay_timer < pr.head_delay_ms)) {
     p.headYaw = p.headYaw + pr.alpha_head * (p.headTarget - p.headYaw);
   }
   ```

2. **离开头区时重置**：进入死区或眼区时，`head_delay_timer = 0`，下次进头区重新计时。

3. **身体与头联动**：`drawBody` 中 `headOffset = headYaw * 4`，身体随头部轻微偏移，增强真实感。

